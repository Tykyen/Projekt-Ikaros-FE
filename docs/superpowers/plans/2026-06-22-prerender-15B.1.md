# Implementační plán — 15B.1 Prerender veřejných stránek

**Spec:** [spec-15B.1-prerender.md](../../arch/phase-15B/spec-15B.1-prerender.md) · **Stav:** 📝 čeká potvrzení → kód · **Datum:** 2026-06-22

**Rozhodnuto:** runtime prerender · self-hosted Chrome sidecar za nginx UA-detekcí · puppeteer-core + alpine Chromium · ready flag `window.__PRERENDER_READY__` · scope jen pipeline (meta až 15B.2).

---

## Kroky

### P1 — Prerender service (`prerender/`)
Samostatný node kontejner, žádná vazba na FE bundle.

- **`prerender/package.json`** — deps: `puppeteer-core`, `express`, `lru-cache`. Type module, start `node index.js`.
- **`prerender/index.js`:**
  - Spustí **jeden** sdílený browser (puppeteer-core, `executablePath` = system chromium, `--no-sandbox`). Reuse přes požadavky (nová `page` per request, browser drží).
  - `GET /*` → `path` = req.url. Cache lookup (LRU, klíč = path, TTL dle skupiny rout).
  - Miss → `page.goto('http://' + FRONTEND_INTERNAL + path, { waitUntil: 'domcontentloaded' })`, pak `page.waitForFunction('window.__PRERENDER_READY__ === true', { timeout: READY_TIMEOUT })`. Vrátí `page.content()` (outerHTML).
  - **Anonymní** — žádné cookies/auth hlavičky se nenastavují (čistý kontext).
  - **Fallback** — timeout/chyba renderu → vrátí to, co je (partial HTML) nebo proxy raw `index.html`; **nikdy 5xx** botovi (graceful degradation, OO6).
  - LRU cap (počet záznamů) + TTL: statické routy (`/`, `/podminky`, `/ikaros/napoveda`) ~24 h; dynamické (`/svet/*`, `/ikaros/clanky/*`, `/ikaros/galerie/*`, `/ikaros/vesmiry`, `/ikaros/novinky`) ~30 min.
  - Env: `FRONTEND_INTERNAL` (default `http://frontend:80`), `PORT` (default 3000), `READY_TIMEOUT` (default 8000), `CHROMIUM_PATH`.
- **`prerender/Dockerfile`** — `node:20-alpine` + `apk add chromium nss freetype harfbuzz ca-certificates ttf-freefont`, `ENV PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser`, `npm ci`, `CMD node index.js`.

**Ověření P1:** `docker build` projde; samostatně `curl localhost:3000/` (proti běžící SPA) vrátí HTML s obsahem.

### P2 — SPA ready flag (`src/app/`)
- Malá komponenta `PrerenderReady` uvnitř `RouterProvider`: `const fetching = useIsFetching();` + `useEffect` — když `fetching === 0` (po prvním doběhnutí), `requestIdleCallback`/krátký timeout → `window.__PRERENDER_READY__ = true`. Idempotentní, žádný UI výstup (`return null`).
- Pro jistotu globální strop: `setTimeout(() => window.__PRERENDER_READY__ = true, 10000)` v main.tsx (kdyby fetching nikdy neutichlo, prerender nečeká věčně — kryje OO6 na straně SPA).

💡 Flag je no-op pro lidi (nikdo ho nečte); existuje jen pro botí render. Žádný dopad na UX.

**Ověření P2:** `npm run build` + `tsc -b` projde; v dev konzoli `window.__PRERENDER_READY__` je `true` po načtení stránky.

### P3 — nginx routing (`default.conf.template` + `Dockerfile`)
- `default.conf.template` (mimo `server {}`, na úrovni http — pozor: nginx:alpine template jede do `conf.d/`, `map` musí být v http kontextu → ověřit, případně samostatný template):
  ```
  map $http_user_agent $is_bot { default 0; "~*(googlebot|seznambot|bingbot|...|facebookexternalhit|twitterbot|discordbot|slackbot|linkedinbot|whatsapp)" 1; }
  map $uri $is_public { default 0; "~^/$" 1; "~^/podminky" 1; "~^/ikaros/(napoveda|novinky|vesmiry|clanky|galerie)" 1; "~^/svet/[^/]+$" 1; }
  ```
- V `location /`: `set $prerender 0; if ($is_bot) { set $prerender "${is_bot}${is_public}"; } if ($prerender = "11") { rewrite . /prerender last; }` → named `location /prerender { internal; proxy_pass http://${PRERENDER_HOST}$request_uri; }`. (Final tvar doladit — vyhnout se „if is evil", jen `set`+`rewrite`.)
- `Dockerfile` (FE) — `NGINX_ENVSUBST_FILTER` rozšířit o `PRERENDER_HOST`, přidat `ENV PRERENDER_HOST="prerender:3000"`.

⚠️ **Past:** `map` direktiva musí být v `http {}`, ne v `server {}`. nginx:alpine bere `templates/*.template` do `conf.d/` (= uvnitř http). Ověřit, že `map` na začátku `conf.d` souboru projde; jinak dát `map` do vlastního `*.template`.

**Ověření P3:** nginx config test `nginx -t` v kontejneru; `curl -A Googlebot` projde na prerender, běžný UA na SPA.

### P4 — compose + deploy
- **`docker-compose.yml`** — služba `prerender` (`build: ./prerender`, `restart: unless-stopped`, bez host portu), `frontend.depends_on: [prerender]` + `frontend.environment.PRERENDER_HOST: ${PRERENDER_HOST:-prerender:3000}`.
- **`deploy.yml`** — do `.env` na serveru přidat `PRERENDER_HOST` (default kryje, takže volitelné). Flow `compose build/up` postaví novou službu sám.

**Ověření P4:** `docker compose up` lokálně → oba kontejnery běží.

### P5 — integrační ověření (spec §6)
- `curl -A "Googlebot" localhost:<port>/` → HTML s obsahem, ne prázdný root.
- `curl -A "Seznambot" .../svet/<public-slug>` → veřejný obsah.
- `curl -A "Googlebot" .../svet/<private-slug>` → bez privátního obsahu (leak-safe).
- `curl .../` (běžný UA) → SPA shell beze změny.
- `curl -A "Googlebot" .../reset-password` → SPA shell (neprerenderuje se).
- 2× stejná path → druhý dotaz rychlejší (cache hit).
